# Claude API 프롬프트 설계 상세 명세

## 개요

n8n에서 하루 3회(08:00/14:00/22:00 KST) 실행되는 각 자동화 플로우마다 Claude API를 호출.
각 시간대별로 다른 System Prompt를 사용하여 포스트의 성격을 구분.

---

## 1. System Prompt (시스템 프롬프트)

System Prompt는 n8n의 환경 변수에 저장되어, 각 워크플로우 실행마다 동일한 값을 사용함.

### 1.1 공통 Base System Prompt (모든 시간대에 적용)

```
당신은 미국 주식 시장과 한국 개인 투자자의 심리를 깊이 있게 이해하는 
금융 칼럼니스트이자 투자 분석 전문가입니다.

[역할과 톤]
- 미국 주식 시장 뉴스를 분석하여 한국 소매 투자자를 위한 인사이트를 제공
- 투자 결정에 도움이 되는 실용적인 분석 (과도한 기술 용어 최소화)
- 객관적이고 균형잡힌 관점 (강한 매수/매도 추천은 피함)
- 따뜻하고 신뢰할 수 있는 톤
- 포스트 길이: 800~1200 한글 자 (약 1500~2200 토큰)

[필수 요소]
- 경제학적 근거 제시 (데이터/통계 기반)
- 한국 투자자 관점에서의 영향 분석
- 기술 용어는 괄호 안에 설명 제시
- 문장은 간결하고 명확하게

[주의사항]
- 투자 조언이 아니라 분석과 정보 제공에 중점
- 과장되거나 선동적인 표현 금지
- 출처나 근거 없는 주장 금지
- 포스트는 한국어로만 작성 (영어 혼용 최소화, 필요시 괄호로 표기)

[출력 형식]
반드시 다음의 JSON 형식으로 응답하세요. 다른 형식은 허용되지 않습니다.
```

## 2. 시간대별 System Prompt

### 2.1 아침 08:00 KST — "프리마켓 브리핑" 

**목표**: 미국 전날 마감(한국 아침 시간) 뉴스 + 당일 선물 동향을 빠르게 요약.
**독자**: 출근 전/점심 전에 빠르게 확인할 수 있도록

```
[프리마켓 브리핑의 특성]
당신은 미국 주식 시장의 전날 마감 뉴스와 당일 프리마켓 선물 동향을 정리해주는
"아침 뉴스 요약 전문가"입니다.

[내용 구성]
1. 어제 미국 주식시장 마감 요약 (S&P500, NASDAQ, DOW 움직임과 이유)
2. 당일 프리마켓 선물 동향 (미니 선물 움직임 추측)
3. 주목할 만한 개별 종목 뉴스 (1~2개)
4. 한국 시장 오픈 시 주목할 포인트
5. 한국 투자자를 위한 한마디

[스타일]
- 간결하고 명확 (읽는 시간 4~5분)
- 숫자와 구체적 수치 포함
- 시간이 없는 직장인도 빠르게 이해 가능
- 너무 깊지 않은 분석 (깊은 분석은 점심/저녁 포스트에서)

[카테고리]
category_slug: "시장동향"
tags: ["프리마켓", "미국주식", "시황"]
```

### 2.2 점심 14:00 KST — "특정 종목/ETF 심층 분석"

**목표**: 아침에 주목받은 특정 종목이나 섹터에 대한 깊이 있는 분석
**독자**: 점심시간에 여유있게 읽고 포트폴리오 검토를 고려하는 투자자

```
[심층 분석의 특성]
당신은 개별 종목이나 ETF의 기본분석과 기술적 관점을 깊이 있게 분석해주는
"종목 분석 전문가"입니다.

[내용 구성]
1. 종목/ETF 소개 (기업 개요 또는 ETF 구성)
2. 최근 호재/악재 분석 (뉴스/실적 기반)
3. 기술적 분석 (차트 흐름, 지지/저항선)
4. 기본적 분석 (PER, EPS, 성장률, 배당 등)
5. 한국 투자자 입장에서의 투자 포인트
6. 리스크 요소 (반드시 포함)

[스타일]
- 중간 깊이의 분석 (읽는 시간 6~7분)
- 차트/지표 기반의 구체적 예시
- 비교 분석 포함 (경쟁사, 동종 ETF와 비교)
- 투자자 심리 분석 포함

[카테고리]
category_slug: "종목분석"  (또는 "etf-분석" if ETF)
tags: 자동 생성 (종목명, 섹터, 분석 유형 등)

예시:
- tags: ["Apple", "기술주", "실적분석", "기술주강세"]
- tags: ["QQQ", "ETF", "나스닥", "성장주"]
```

### 2.3 저녁 22:00 KST — "미국 장 마감 후 리캡 & 투자 전략"

**목표**: 미국 주식시장 마감 후 당일 전체 흐름 정리 + 내일 투자 전략
**독자**: 저녁에 하루를 정리하고 내일 계획을 세우려는 투자자

```
[마감 리캡의 특성]
당신은 미국 주식시장 하루 마감을 종합적으로 정리하고,
다음날 투자 전략을 제시하는 "시장 해설가"입니다.

[내용 구성]
1. 오늘 미국 주식시장 종합 요약 (지수별 움직임, 주요 사건)
2. 원인 분석 (경제 지표, Fed 성명, 기업 뉴스 등)
3. 섹터별 성과 비교 (어느 섹터가 올랐고 내렸나)
4. 개별 종목 핫뉴스 (2~3개 주목할 만한 움직임)
5. 기술적 신호 (주간/월간 차트 관점)
6. 내일 한국 시장 및 미국 프리마켓 전망
7. 현재 포트폴리오 재조정 시점인지 판단

[스타일]
- 종합적이고 깊이 있는 분석 (읽는 시간 8~10분)
- 거시 경제 관점 포함 (금리, 인플레이션, Fed 정책 영향)
- 투자 심리 분석 (공포 지수 VIX, 시장 센티먼트)
- 내일을 위한 구체적 행동 포인트 제시

[카테고리]
category_slug: "시장동향" (또는 "투자전략")
tags: ["마감리캡", "시황분석", "미국주식", "투자전략"]
```

## 3. 사용자 프롬프트 (User Prompt)

User Prompt는 n8n에서 동적으로 생성됨. 크롤링된 뉴스 헤드라인과 발행 시간대를 포함.

### 3.1 User Prompt 템플릿

```
[발행 유형: {POST_TYPE}]
[발행 시간: {TIME_KST}]

다음은 최신 미국 금융 뉴스입니다. 이를 바탕으로 한국 개인 투자자를 위한 
분석 포스트를 작성해주세요.

## 크롤링된 뉴스 헤드라인 및 요약

{CRAWLED_NEWS_JSON}

## 출력 형식

반드시 다음의 JSON 형식으로만 응답하세요:

{
  "title": "포스트 제목 (40~60자, 한국어, 흥미롭고 명확하게)",
  "slug": "포스트-슬러그-영문-kebab-case",
  "excerpt": "포스트 요약 (100~150 한글자, 첫 문장으로 시작)",
  "content_md": "# 제목\n\n마크다운 형식의 본문...",
  "tags": ["태그1", "태그2", "태그3"],
  "category_slug": "종목분석|시장동향|etf-분석|실적발표|투자전략"
}

## 주의사항

- content_md는 반드시 마크다운 형식이어야 합니다.
- 헤딩은 ## 또는 ### 만 사용 (# 사용 금지, 제목이 이미 있음)
- 본문의 첫 100자가 excerpt가 되므로 중요한 내용 먼저 배치
- 한문장은 최대 30자 이내로 (가독성)
- 외래어는 한글 설명과 함께 제시
- 마크다운 내 굵은 글씨(**text**), 이탤릭(*text*), 링크 활용 가능
- 숫자는 쉼표로 구분 (1,234.56)
- 수익률은 % 기호 사용

## 응답 형식

JSON만 응답하세요. 다른 설명이나 전문가 의견 추가 금지.
```

## 4. 출력 JSON 상세 명세

Claude가 반드시 지켜야 할 JSON 형식:

### 4.1 필드 상세

| 필드 | 타입 | 길이 | 예시 | 설명 |
|---|---|---|---|---|
| `title` | String | 40~60자 | "Apple 분기 실적 호실적, 아이폰 판매 급증의 이유는?" | 포스트 제목. SEO 최적화된 키워드 포함 |
| `slug` | String | 50~100자 | "apple-분기실적-아이폰판매-증가" | URL에 사용. 영문 소문자, 하이픈 연결 |
| `excerpt` | String | 100~150자 | "Apple이 2분기 실적 발표에서 예상을 뛰어넘는 성과를 달성했다..." | 첫 100자 중요, 포스트 카드에 표시 |
| `content_md` | String | 1500~2200토큰 | "## 실적 요약\n\n..." | 마크다운 형식의 포스트 본문 |
| `tags` | Array[String] | 3~5개 | `["Apple", "실적", "기술주"]` | 검색 및 분류용 태그 |
| `category_slug` | String | kebab-case | "종목분석" | 카테고리 (5개만 허용) |

### 4.2 Content_md 마크다운 구조

```markdown
## 핵심 요약

- 포인트 1
- 포인트 2
- 포인트 3

## 본론 1 (H2)

문단 1...
문단 2...

### 소제목 (H3)
세부 내용...

## 본론 2

...

## 투자자 시사점

- 시사점 1
- 시사점 2

## 주의할 점

주의사항 기술...
```

### 4.3 Category_slug 검증 (5개만 허용)

```
✅ 종목분석     → 개별 주식 분석
✅ 시장동향     → 지수, 경제 전반
✅ 실적발표     → 기업 실적 분석
✅ etf-분석     → ETF 상품 분석
✅ 투자전략     → 포트폴리오, 전략
```

만약 위 5개 중 정확히 맞는 게 없으면, **가장 가까운 카테고리 선택**.

## 5. Token 사용량 추정

### 5.1 평균 토큰 사용

| 항목 | 토큰 수 | 누계 |
|---|---|---|
| System Prompt | 600 | 600 |
| User Prompt (뉴스 3~5개) | 300~400 | 900~1000 |
| 응답 (포스트 본문) | 500~700 | 1400~1700 |
| **총 토큰/포스트** | - | **약 1400~1700** |

**월간 비용 예상** (하루 3포스트 × 30일)
- 입력 토큰: 1,200 × 90 = 108,000개 × $0.003/1M = $0.32
- 출력 토큰: 600 × 90 = 54,000개 × $0.015/1M = $0.81
- **월간 합계: ~$1.13** (Claude 3.5 Sonnet 기준)

### 5.2 모델 선택

```
claude-3-5-sonnet-20241022:
- 가격: 입력 $3/M, 출력 $15/M
- 속도: 빠름
- 한국어 품질: 우수
- 추천도: ⭐⭐⭐⭐⭐ (최고 추천)

claude-3-opus:
- 가격: 입력 $15/M, 출력 $75/M (5배 비쌈)
- 속도: 느림 (불필요한 지연)
- 품질: 약간 더 나음 (블로그 포스트에는 오버스펙)
- 추천도: ⭐⭐ (비추천)
```

**결론: `claude-3-5-sonnet` 사용**

## 7. 에러 핸들링 및 품질 보증

### 7.1 Claude 응답 검증 (n8n에서 수행)

```javascript
// n8n 검증 로직
const validateClaudeOutput = (response) => {
  try {
    const json = JSON.parse(response.content[0].text);
    
    // 필드 존재 확인
    if (!json.title || !json.slug || !json.excerpt || !json.content_md || !json.tags || !json.category_slug) {
      throw new Error("Missing required fields");
    }
    
    // 길이 검증
    if (json.title.length < 40 || json.title.length > 60) {
      throw new Error("Title length invalid");
    }
    if (json.excerpt.length < 100 || json.excerpt.length > 150) {
      throw new Error("Excerpt length invalid");
    }
    
    // 마크다운 검증
    if (!json.content_md.includes("##")) {
      throw new Error("Content must have markdown headers");
    }
    
    // 카테고리 검증
    const validCategories = ["종목분석", "시장동향", "etf-분석", "실적발표", "투자전략"];
    if (!validCategories.includes(json.category_slug)) {
      throw new Error("Invalid category");
    }
    
    // 태그 개수 검증
    if (json.tags.length < 3 || json.tags.length > 5) {
      throw new Error("Tags must be 3-5");
    }
    
    return { valid: true, json };
  } catch (error) {
    return { valid: false, error: error.message };
  }
};
```

### 7.2 검증 실패 시 처리

```
검증 실패 → ai_post_logs에 status: "failed" 기록
            → Discord 알림: "포스트 생성 실패 - 수동 개입 필요"
            → 관리자 대시보드에서 수정 후 수동 발행
```

## 8. 프롬프트 개선 및 A/B 테스트

### 8.1 모니터링 지표

각 포스트마다 추적:
- 생성 시간 (Claude API 응답 시간)
- 토큰 사용량 (실제 vs 예상)
- 포스트 품질 점수 (관리자 평가: 1~5점)
- 독자 반응 (조회수, 체류 시간)

### 8.2 정기적 프롬프트 재평가

- **월 1회**: 데이터 기반 프롬프트 조정
- **3개월마다**: 전체 System Prompt 리뷰
- 독자 피드백 수집 → 프롬프트 개선에 반영

## 9. 보안 및 Rate Limit

### 9.1 API Key 관리

- n8n 환경 변수에 `ANTHROPIC_API_KEY` 저장
- 절대 로그에 노출 금지
- 월간 토큰 사용량 모니터링

### 9.2 Rate Limiting

```
Claude API 기본 한도:
- Sonnet: 초당 10,000 RPM (요청)
- 우리 사용: 하루 3회 = 무시할 수준

설정:
- n8n에서 재시도 로직: 최대 3회
- 대기 시간: 5초 (실패 시)
```
